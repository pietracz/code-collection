EBS Volume
Elastic Block Store ist ein Netzwerkspeicher zum zuschalten von zusätzlichen Speicher-Volumen,
welcher an eine EC2 Instanz angeknüpft werden kann. Die Anwendungsfälle variieren stark,
da es abhängig davon ist, was wir mit der EC2 Instanz machen wollen.
Beispiele sind Web-Applikationen, Datenbanken anlegen, Datenbanken analysieren,
Betriebssystem und Zusatzdaten speichern - also quasi alles, was eine normale Festplatte machen kann.
EBS müssen mit der EC2 Instanz in einer Availability Zone sein, darüber hinaus kann eine EBS nur unter bestimmten Bedingungen
an eine zweite EC2 Instanz angeknüpft werden - ansonsten grundsätzlich nicht.
Die IOPS bestimmen die Geschwindigkeit der Datenverarbeitung (Input/Output Operations Per Second)

EBS Snapshots 
Ist ein Backup einer EBS, zu einem bestimmten Zeitpunkt
Snapshots können in andere Regionen oder AZs übertragen werden.
Darüber hinaus werden Snapshots in einem S3 Bucket abgespeichert, sollten diese weniger oft benötigt werden,
können die Snapshots im Snapshot Archiv abgelegt werden (75% Kostensenkung)
Des Weiteren gibt es auch einen Snapshot Recycle Bin, dieser muss erst eingerichtet werden, um vor unbeabsichtigter Löschung
vorzubeugen.

Amazon Machine Images (AMI)
Ein Abbild kann von einer EC2 Instanz erzeugt werden.
Es gibt einen Image Builder, der auf Grundlage von Einstellungen Images erstellen kann. Automatisiert.
Diese Images können dann wiederum von einem anderen Service genutzt werden, um automatisiert zu skalieren.

Ec2 Instance Store
Der EC2 Instanz Speicher ist der Speicher, der physisch mit der Maschine verbunden ist und ist preislich in der
gewöhnlichen Instanz enthalten. Dies ist aber lediglich ein temporärer Speicher für Daten, die sich häufig und
schnell ändern könnten - z.B. Caches und Arbeitsdaten. Dadurch, dass er in der selben Maschine ist, bietet er eine
geringere Latenz. Heißt er ist nicht über das Netzwerk an die Maschine angeknüpft, sondern über die Hardware direkt.
Sie erfüllen eine ähnliche Funktion wie ein RAM, aber sind Festplatten. Hier ist der kleinste Werte der IOPS 35.000
Heißt, dass Daten schnell verarbeitet werden können.

EFS
EFS klingt in der Nutzung fast wie EBS, nur wesentlich stärker.
EFS oder auch Elastic file system ist ein vollständig verwalteter Service, der automatisch Speicher zur Verfügung stellt,
ohne dabei den Ablauf zu behindern. Über EC2 kann dann wohl auch auf den Speicher,
der durch EFS zur Verfügung gestellt wird, zugegriffen werden.
